# v0.8.0 - External GGUF PoC Planning

## أبرز الميزات
- **محرك استدلال مستقر:** تم الانتقال إلى الإصدار الرسمي والمستقر من LiteRT API (v1.0.1) لضمان أقصى درجات التوافق مع معمارية أندرويد الحديثة وحل مشاكل تعارض المكتبات.
- **إعادة هيكلة معمارية شاملة:** دمج Dagger Hilt لإدارة التبعيات وضمان استقرار النظام.
- **إدارة ذكية للذاكرة:** تطبيق نمط التحميل الكسول (Lazy Loading) في `ModelManager` مع الاستجابة الحية لتنبيهات النظام (onTrimMemory) لتفادي الإغلاق المفجبئ.
- **تحسين مسار البث (Streaming):** تقليل استهلاك البطارية والمعالج بنسبة كبيرة عبر تجميع التوكنات (Buffering) واستخدام واجهات Compose المستقرة (@Immutable).
- **محرك RAG مطور:** تقطيع دقيق للنصوص يدعم الترقيم العربي مع خاصية التداخل (Overlap) لضمان جودة الإجابات.
- **تمرير ذكي (Smart Scroll):** تحسين واجهة المحادثة لتدعم التمرير التلقائي الذكي دون مقاطعة المستخدم أثناء القراءة.
- إضافة خطة تجربة GGUF خارج تطبيق نبض لضمان عزل المخاطر.
- إضافة قالب موحد لتسجيل نتائج الـ PoC (Proof of Concept) للمقارنة العادلة.
- تحديد بيئات الاختبار الخارجية المتاحة:
  - Termux على Android (قياس الأداء الحقيقي للعتاد).
  - مشروع Android منفصل (محاكاة دمج JNI).
  - Linux Desktop (اختبار الجودة اللغوية السريعة).
- تحديد معايير القياس التقنية بدقة:
  - TTFT (Time To First Token)
  - TPS (Tokens Per Second)
  - Peak RAM (أقصى استهلاك للذاكرة)
  - Crash / ANR (مراقبة الاستقرار)
  - Streaming & Stop Generation (دعم الوظائف الحيوية)
- تحديد Prompts اختبار الجودة والسلامة المهنية ومنع الهلوسة.
- إبقاء **Qwen** كنموذج افتراضي ومستقر وحيد حالياً.
- الالتزام بعدم إدخال JNI أو llama.cpp أو أي مكتبات Native إلى التطبيق الأساسي في هذه المرحلة.

## إصلاحات الأخطاء
- إصلاح مشكلة "Vision backend constraint mismatch" التي كانت تمنع تشغيل نماذج الرؤية (Gemma 3) على معالج الرسوميات (GPU). تم تحديث محرك LiteRT-LM ليقوم بضبط Backend الرؤية ديناميكياً ليطابق المحرك الأساسي.

## القرار الهندسي
تبدأ تجربة GGUF خارج تطبيق نبض أولاً كـ Sandbox، وبناءً على النتائج الميدانية سيتم اتخاذ قرار لاحق:
- **Proceed:** البدء في تصميم الدمج التجريبي داخل Dev Mode.
- **Delay:** تأجيل الفكرة لمزيد من البحث.
- **Reject:** صرف النظر عن GGUF إذا تبين عدم ملاءمته تقنياً.
