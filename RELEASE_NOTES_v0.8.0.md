# v0.8.0 - External GGUF PoC Planning

## أبرز الميزات
- إضافة خطة تجربة GGUF خارج تطبيق نبض لضمان عزل المخاطر.
- إضافة قالب موحد لتسجيل نتائج الـ PoC (Proof of Concept) للمقارنة العادلة.
- تحديد بيئات الاختبار الخارجية المتاحة:
  - Termux على Android (قياس الأداء الحقيقي للعتاد).
  - مشروع Android منفصل (محاكاة دمج JNI).
  - Linux Desktop (اختبار الجودة اللغوية السريعة).
- تحديد معايير القياس التقنية بدقة:
  - TTFT (Time To First Token)
  - TPS (Tokens Per Second)
  - Peak RAM (أقصى استهلاك للذاكرة)
  - Crash / ANR (مراقبة الاستقرار)
  - Streaming & Stop Generation (دعم الوظائف الحيوية)
- تحديد Prompts اختبار الجودة والسلامة المهنية ومنع الهلوسة.
- إبقاء **Qwen** كنموذج افتراضي ومستقر وحيد حالياً.
- الالتزام بعدم إدخال JNI أو llama.cpp أو أي مكتبات Native إلى التطبيق الأساسي في هذه المرحلة.

## القرار الهندسي
تبدأ تجربة GGUF خارج تطبيق نبض أولاً كـ Sandbox، وبناءً على النتائج الميدانية سيتم اتخاذ قرار لاحق:
- **Proceed:** البدء في تصميم الدمج التجريبي داخل Dev Mode.
- **Delay:** تأجيل الفكرة لمزيد من البحث.
- **Reject:** صرف النظر عن GGUF إذا تبين عدم ملاءمته تقنياً.
