# مقارنة محركات تشغيل GGUF على أندرويد - GGUF Android Backend Comparison (v0.7.0)

يهدف هذا المستند إلى تقييم الخيارات التقنية المتاحة لدمج دعم صيغة **GGUF** في تطبيق "نبض"، لضمان اختيار المسار الأكثر استقراراً وكفاءة.

## الخيارات التقنية للمقارنة

### 1. llama.cpp عبر واجهة JNI (الدمج المباشر)
يتم بناء مكتبات `llama.cpp` الأصلية باستخدام NDK وربطها بـ Kotlin عبر Java Native Interface.

|---|---|
| **سهولة الدمج** | صعبة (تتطلب إدارة كود C++ و JNI) |
| **دعم GGUF** | ممتاز (الدعم الرسمي والأحدث) |
| **دعم ARM64** | ممتاز |
| **حجم المكتبات** | كبير (يزيد حجم APK بمقدار 20-40 ميجابايت) |
| **الأداء المتوقع** | عالٍ جداً (استغلال كامل للـ Hardware) |
| **استهلاك الذاكرة** | مرتفع (يحتاج إدارة يدوية دقيقة) |
| **دعم Streaming** | نعم |
| **دعم إيقاف التوليد** | نعم |
| **مخاطر Crash / ANR** | عالية (بسبب طبيعة الكود الأصلي) |

## المسار الموصى به (Recommended Path)

بناءً على التحليل أعلاه، نوصي بالآتي:
1. **لا يتم دمج GGUF أو JNI مباشرة في تطبيق نبض الآن.**
2. البدء بإنشاء **مشروع تجريبي منفصل (GGUF PoC)** لاختبار مكتبة `llama.cpp` وقياس استهلاكها للموارد.
3. الحفاظ على **Qwen** كنموذج افتراضي ومستقر وحيد في النسخة الأساسية.
4. أي دمج مستقبلي لـ GGUF سيكون حصرياً داخل **Dev Mode** ولن يظهر للمستخدم العادي حتى يثبت استقراره التام.

## الحد الأدنى للتجربة (Minimum Viable GGUF PoC)
تتضمن التجربة الناجحة خارج التطبيق:
- تشغيل نموذج GGUF صغير جداً (مثل Qwen-0.5B-GGUF).
- العمل على جهاز أندرويد بمعمارية ARM64.
- قياس سرعة أول رد (TTFT) وسرعة التوليد (TPS).
- مراقبة أقصى استهلاك للذاكرة (Peak RAM).
- تسجيل كافة الانهيارات (Crashes) وأسبابها.

## بوابات القرار (Decision Gates)
لن يتم الانتقال لمرحلة الدمج الفعلي إلا إذا:
- عمل الـ PoC لمدة 48 ساعة دون انهيارات (Zero Crash).
- كان حجم المكتبات مبرراً بالقيمة المضافة.
- تم التأكد من دعم الـ Streaming والإيقاف الفوري.
- لم يتطلب الدمج تغييرات جذرية في هيكل `ChatViewModel` الحالي.

## Do Not Implement Yet
> **تنبيه أمان:**
> - لا تضف JNI أو llama.cpp في هذا الالتزام.
> - لا تضف ملفات مكتبات أصلية (.so).
> - لا تشغل MiniCPM أو أي نموذج GGUF داخل نبض الآن.
> - لا تغير إعدادات Qwen المستقرة.
