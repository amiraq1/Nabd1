# v0.5.0 - Model Runtime Experiments

## أبرز الميزات
- إضافة خطة تجارب النماذج المحلية لتقييم الأداء والجودة بشكل منهجي.
- إضافة تصميم تقني شامل لمقارنة النماذج والاعتمادات.
- إضافة `ModelRuntimeModels` لتمثيل النماذج المرشحة ونتائج الجودة والأداء برمجياً.
- إضافة `ModelRuntimeEvaluator` لتقييم أهلية النماذج بناءً على معايير صارمة.
- إضافة قالب موحد (`MODEL_RUNTIME_EVALUATION_TEMPLATE.md`) لتسجيل نتائج التقييم.
- إضافة واجهة مطور داخلية (`ModelRuntimeDevScreen`) لعرض مرشحي النماذج وحالاتهم.
- تسجيل **MiniCPM** كمرشح تجريبي فقط لبدء عملية التقييم.
- إبقاء **Qwen** كنموذج افتراضي مستقر وحيد للتطبيق.

## MiniCPM
MiniCPM مسجل كمرشح تجريبي فقط في هذا الإصدار:
- لا يعمل فعلياً داخل التطبيق (يتطلب دمج محرك تشغيل GGUF مستقبلاً).
- لا يمكن للمستخدم النهائي اختياره أو تفعيله.
- لا يستبدل Qwen بأي حال من الأحوال.
- يحتاج إلى اختبارات جودة وأداء مكثفة قبل التفكير في أي اعتماد.

## قواعد الأمان
- لا يتم تغيير النموذج الافتراضي تلقائياً.
- Qwen يبقى هو خيار الرجوع الآمن دائمًا.
- أي نموذج جديد يجب أن يجتاز اختبارات v0.3.0 و v0.4.0 بالكامل.
- لا يُعتمد أي نموذج بمتوسط تقييم أقل من 4.5/5.
- لا يُعتمد أي نموذج يظهر فشلاً في معايير السلامة أو مقاومة الهلوسة.

## الحالة
Stable (إصدار مستقر لبنية التجارب)
