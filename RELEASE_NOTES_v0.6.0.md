# v0.6.0 - Runtime Abstraction Foundation

## أبرز الميزات
- إضافة خطة تقنية شاملة لتجربة GGUF Runtime مستقبلاً لتعزيز مرونة التطبيق.
- إضافة تصميم تقني موحد لتجريد محركات التشغيل وفصل منطق المحادثة عن الاستدلال.
- إضافة `ModelRuntimeInterface` كواجهة برمجية موحدة لكافة محركات التشغيل القادمة.
- إضافة نماذج البيانات الأساسية: `RuntimeGenerationRequest`, `RuntimeGenerationResult`, و `RuntimeGenerationChunk`.
- إضافة `RuntimeMetrics` لقياس مؤشرات الأداء الحيوية (TTFT, TPS, Memory Usage) بشكل معياري.
- إضافة `RuntimeStatus` و `RuntimeError` لضمان إدارة احترافية لدورة حياة المحرك ومعالجة الأخطاء.
- إضافة `QwenRuntimeAdapter` كـ Stub آمن وتأسيسي لربط المحرك المستقر بالهيكل الجديد.
- إضافة `GgufRuntimeAdapter` كـ Stub تجريبي لتمهيد الطريق لدعم نماذج GGUF.
- تجهيز البنية التحتية المستقبلية لدعم صيغ نماذج متعددة (مثل MiniCPM) دون المساس بالاستقرار الحالي.

## ما لا يحتويه هذا الإصدار
- لا يوجد تشغيل GGUF فعلي (يتطلب دمج مكتبات Native مستقبلاً).
- لا يوجد JNI أو كود C++ مدمج.
- لا يوجد llama.cpp أو ملفات مكتبات أصلية (.so).
- لا يوجد تبديل محركات (Runtime Switching) متاح للمستخدم النهائي.
- لا يوجد تشغيل لنماذج MiniCPM داخل التطبيق في هذه المرحلة.
- لا يوجد أي تغيير في آلية تشغيل Qwen الحقيقية؛ التطبيق مستمر بالاعتماد على المحرك النشط المستقر.

## الحالة
Stable Foundation (بنية تحتية مستقرة وجاهزة للتطوير)
