# التصميم التقني لمحرك التحقق - Verification Engine Technical Design (v0.4.0)

يوضح هذا المستند الهيكل البرمجي المقترح لنظام التحقق والمصادر داخل تطبيق "نبض". الهدف هو تحويل القواعد النظرية في خطة العمل إلى مكونات برمجية قابلة للتنفيذ.

## 1. تعريفات البيانات الأساسية (Core Definitions)

تم اقتراح الهياكل التالية باستخدام لغة Kotlin لتمثيل حالات التحقق:

```kotlin
/**
 * يحدد مستوى التحقق المطلوب بناءً على نوع السؤال.
 */
enum class VerificationLevel {
    LEVEL_0_DIRECT,               // أسئلة عامة، مفاهيم، برمجة
    LEVEL_1_CONTEXTUAL_CAUTION,   // أسئلة تاريخية، رياضية قديمة، أرقام
    LEVEL_2_RECENT_OR_SENSITIVE   // أخبار، أسعار، مسؤولون، صحة، مال، قانون
}

/**
 * يحدد نوع المصدر المطلوب لدعم الإجابة.
 */
enum class SourceRequirement {
    NONE,                   // لا يتطلب مصدر
    STATIC_KNOWLEDGE,       // معرفة النموذج الثابتة كافية
    RECENT_SOURCE,          // يتطلب مصدر إخباري حديث
    USER_PROVIDED_SOURCE,   // يعتمد على مستندات المستخدم (RAG)
    PROFESSIONAL_ADVICE     // يتطلب استشارة مختص مهني
}

/**
 * يمثل الحالة النهائية لدرجة التحقق من الإجابة.
 */
enum class VerifiedAnswerState {
    VERIFIED,               // معلومة مؤكدة
    PARTIALLY_VERIFIED,     // مؤكدة جزئياً
    UNVERIFIED,             // غير مؤكدة (تحذير)
    NEEDS_RECENT_CHECK,     // تتطلب تحقق من الإنترنت
    SAFETY_RESTRICTED       // مقيدة لأسباب تتعلق بسلامة المحتوى
}

/**
 * القرار النهائي لمحرك التحقق حول كيفية التعامل مع المدخلات.
 */
data class VerificationDecision(
    val level: VerificationLevel,
    val sourceRequirement: SourceRequirement,
    val answerState: VerifiedAnswerState,
    val shouldAnswerDirectly: Boolean,
    val shouldAddCaution: Boolean,
    val shouldAskForSource: Boolean,
    val reason: String
)
```

## 2. منطق التصنيف (VerificationClassifier Logic)

يعمل هذا المكون على تحليل النص المدخل (Input Analysis) لتصنيفه وفق القواعد التالية:

### المستوى 0 (Level 0):
- **المجالات:** شرح مفاهيم، كتابة نصوص إبداعية، مساعدة برمجية عامة، تلخيص نصوص.
- **القاعدة:** إجابة مباشرة بدون قيود زمنية.

### المستوى 1 (Level 1):
- **المجالات:** تواريخ قديمة، بطولات رياضية سابقة، أرقام تاريخية، إحصائيات سكانية سابقة.
- **القاعدة:** البحث عن مؤشرات زمنية (سنة، موسم) وإضافة تنبيهات حول احتمالية التباس التواريخ.

### المستوى 2 (Level 2):
- **المجالات:** الأخبار العاجلة، أسعار العملات/الأسهم، قوانين حديثة، إصدارات تقنية جديدة، مسؤولون حاليون، نتائج مباريات اليوم/أمس، استشارات طبية/مالية/قانونية.
- **القاعدة:** منع الجزم بالمعلومة والمطالبة بمصدر حديث أو توجيه المستخدم لمختص.

## 3. بناء المطالبة الذكي (VerificationPromptBuilder)

يقوم هذا المكون ببناء تعليمات إضافية يتم دمجها في الـ System Prompt بناءً على قرار التحقق:

- **لحالة Level 0:**
  > "أجب مباشرة وبوضوح وبأسلوبك المعتاد."
- **لحالة Level 1:**
  > "أجب بحذر. انتبه لأي التباس محتمل بين السنة والموسم أو التواريخ الرسمية، ووضح ذلك للمستخدم."
- **لحالة Level 2:**
  > "لا تجزم بالمعلومة دون تحقق حديث أو مصدر موثوق. إذا لم يتوفر مصدر، صرّح بوضوح بأن المعلومة قديمة أو تحتاج لتحقق إخباري حديث."
- **لحالة PROFESSIONAL_ADVICE:**
  > "ارفض تقديم نصيحة طبية أو مالية أو قانونية حاسمة. قدم معلومات عامة فقط وحث المستخدم على استشارة مختص."
