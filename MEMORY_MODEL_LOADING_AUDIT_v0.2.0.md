# تقرير فحص تحميل النماذج وإدارة الذاكرة (v0.2.0)

تم إجراء فحص دقيق للكود المصدري لمشروع "نبض" للتأكد من كفاءة استهلاك الذاكرة عند التعامل مع ملفات النماذج الكبيرة (.litertlm, .tflite).

## 1. نتائج الفحص التقني

### أ. نماذج المحادثة (.litertlm)
- **الآلية:** يتم تمرير مسار الملف الكامل `modelPath` كرسلسلة نصية (String) إلى محرك الاستدلال `LiteRtLmInferenceEngine`.
- **موقع الكود:** `LiteRtLmInferenceEngine.kt` (الوظيفة `load`).
- **النتيجة:** المحرك الأصلي (Native Library) هو المسؤول عن فتح الملف، مما يضمن كفاءة عالية وتجنب استهلاك الذاكرة في طبقة Java/Kotlin.

### ب. نموذج التضمين (.tflite)
- **الآلية:** يتم استخدام رسم الذاكرة الخرائطي (**Memory Mapping - mmap**) عبر `FileChannel.map`.
- **موقع الكود:** `EmbeddingEngine.kt` (الوظيفة `mapModelFile`).
- **النتيجة:** يتم تحميل النموذج في الذاكرة الافتراضية للنظام دون استهلاكه من مساحة الـ JVM Heap، مما يمنع حدوث `OutOfMemoryError` عند التعامل مع نماذج التضمين الكبيرة.

### ج. استيراد النماذج والتحقق من النزاهة
- **الاستيراد:** يتم استخدام `input.copyTo(output)` مع وجود بفر (Buffering) تلقائي، مما يجعل عملية نسخ الملفات آمنة تماماً مهما كان حجمها.
- **التحقق (SHA-256):** يتم استخدام `ByteArray(8192)` فقط كبفر صغير للقراءة التدفقية، ولا يتم تحميل الملف بالكامل في الذاكرة لحساب البصمة.
- **موقع الكود:** `ModelManager.verifyModelIntegrity` و `EmbeddingModelManager.importEmbeddingModel`.

## 2. ممارسات الذاكرة المكتشفة
- **لا يوجد استخدام لـ `readBytes()`**: تم التأكد من عدم وجود أي استدعاءات تقوم بتحويل ملفات النماذج إلى مصفوفات بايتات كاملة في الذاكرة.
- **تحرير الموارد**: يتم استدعاء `unload()` و `close()` للمحركات القديمة قبل تحميل محركات جديدة لضمان تحرير الذاكرة الرسومية (GPU) والذاكرة العشوائية.

## 3. التوصيات المستقبلية
1. **الالتزام بالمسار**: الاستمرار دائماً في تمرير `modelPath` أو استخدام `mmap` وعدم محاولة تحميل أي نموذج كـ `ByteArray`.
2. **مراقبة RAM**: الاعتماد على واجهة أداء نبض (Performance UI) لمراقبة استهلاك الذاكرة العشوائية الفعلي أثناء التحميل.
3. **وقت التحميل**: مراقبة Latency التحميل؛ إذا زاد بشكل كبير، قد يشير ذلك إلى بطء في الـ I/O أو مشاكل في الـ Memory Mapping على بعض الأجهزة.
