# واجهة تجارب تشغيل النماذج للمطورين - Model Runtime Dev UI (v0.5.0)

تعد هذه الواجهة أداة داخلية مخصصة للمطورين والمختبرين لمتابعة وتقييم النماذج اللغوية المحلية المرشحة للعمل داخل تطبيق "نبض".

## هدف الشاشة
الهدف الأساسي هو توفير رؤية شاملة للمرشحين (Model Candidates) وحالاتهم التقنية دون التأثير على تجربة المستخدم النهائي أو المساس باستقرار النموذج الافتراضي.

## ما تعرضه الشاشة
- **قائمة النماذج:** تعرض النماذج المعرفة في النظام (مثل Qwen و MiniCPM).
- **البيانات التقنية:** تشمل معرف النموذج (Model ID)، نوع محرك التشغيل (Runtime)، والامتداد المدعوم.
- **الحالة الحالية:** توضح ما إذا كان النموذج مستقراً (Stable)، تجريبياً (Experimental)، أو مرفوضاً (Rejected).
- **الملاحظات الفنية:** تعرض معلومات إضافية حول دور كل نموذج في الخطة التطويرية.
- **أدوات المطور:** أزرار لنسخ معرف النموذج وعرض تفاصيل التقييم (نظرياً في هذه المرحلة).

## ما لا تفعله الشاشة (Safety Limits)
- **لا يوجد تبديل للنماذج:** الشاشة لا تحتوي على أي خيار لتغيير النموذج الافتراضي الفعلي للتطبيق.
- **لا يوجد تشغيل للنماذج:** الشاشة لا تقوم بتحميل أو تشغيل أي نموذج تجريبي (MiniCPM) في هذه المرحلة.
- **لا يوجد تأثير على الإعدادات:** الدخول لهذه الشاشة أو التفاعل معها لا يغير أي قيم في `SharedPreferences` أو قواعد البيانات.

## قواعد الأمان (Safety Rules)
1. **Developer Only:** هذه الشاشة مخصصة لأغراض التطوير فقط ويجب ألا تظهر للمستخدم النهائي.
2. **Qwen Immortality:** يبقى Qwen هو النموذج الافتراضي الوحيد القابل للتشغيل.
3. **Soft Integration:** تم دمج الشاشة بشكل "ناعم" (Soft Link) بحيث لا تكسر أي تدفق حالي للمحداثات.

## كيفية الوصول (مستقبلاً)
سيتم ربط هذه الشاشة عبر مدخل مخفي في شاشة الإعدادات (مثل الضغط المطول على رقم الإصدار) أو عبر قائمة خيارات المطور.
