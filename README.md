# نبض - Nabd

نبض هو مساعد ذكاء اصطناعي محلي يعمل على Android بدون خادم خارجي.

## المميزات الرئيسية

- واجهة **Soft UI** عصرية وناعمة مع دعم كامل للـ RTL
- إعداد النموذج بلمسة واحدة عبر معالج (Wizard) مؤتمت
- تشغيل نماذج LiteRT-LM محليًا على الجهاز
- دعم Gemma 3 Multimodal (الرؤية والمحادثة)
- دعم Gemma E2B للمهام اليومية
- دعم Gemma E4B للمهام الثقيلة
- محادثات متعددة مثل ChatGPT
- مكتبة مستندات محلية
- تحليل ملفات PDF والصور
- إجابات من المستندات مع مصادر
- أدوات هاتف آمنة
- ذاكرة نبض محلية اختيارية
- مراقبة الأداء (RAM, TPS) لحظياً
- Material Design 3

## الخصوصية

- المعالجة تتم داخل الجهاز
- لا يتم إرسال المحادثات إلى خادم خارجي من داخل التطبيق
- أداة الخريطة تفتح تطبيق خرائط خارجي بعد تأكيد المستخدم
- سياسة الخصوصية: [PRIVACY_POLICY.md](PRIVACY_POLICY.md)

## المتطلبات

- جهاز Android حديث
- ذاكرة RAM جيدة، ويفضل 8GB أو أكثر للنماذج الثقيلة
- مساحة تخزين كافية لملفات النماذج

## ملاحظة حول النماذج

- النماذج غير مدمجة داخل التطبيق
- يجب استيراد ملفات `.litertlm` يدويًا داخل نبض
- لا ترفع ملفات النماذج إلى GitHub أو إلى المستودع

## النماذج المقترحة

- Gemma E2B
- Gemma E4B

## طريقة الاستخدام

1. ثبّت ملف APK.
2. افتح نبض.
3. اضغط على **"استيراد نموذج وتشغيله"**.
4. اختر ملف الـ `.litertlm` الخاص بك.
5. انتظر ثوانٍ حتى يكتمل المعالج المؤتمت.
6. ابدأ المحادثة مباشرة.

## ما الذي يمكن لنبض فعله؟

- الدردشة المحلية مع نماذج LiteRT-LM
- إدارة محادثات متعددة
- قراءة الصور واستخراج النص منها عبر OCR
- قراءة ملفات PDF وتحليلها
- حفظ المستندات محليًا واستخدامها في الأسئلة اللاحقة
- تشغيل أدوات هاتف آمنة بعد تأكيد المستخدم
- ذاكرة نبض: ذاكرة محلية اختيارية تحفظ فقط المعلومات التي يطلب المستخدم حفظها صراحة، مثل التفضيلات أو الاسم أو ملاحظات بسيطة. يمكن عرضها أو تعطيلها أو مسحها من الإعدادات.

## الأخطاء الشائعة والحلول

### 1. النموذج لا يعمل
- **السبب:** ملف النموذج غير مستورد، أو الملف خاطئ، أو ذاكرة الجهاز غير كافية.
- **الحل:** استورد ملف `.litertlm` الصحيح، ابدأ بتجربة نموذج Gemma E2B أولاً، أعد تشغيل نبض، وأغلق التطبيقات الأخرى لتوفير الذاكرة.

### 2. التطبيق بطيء في الرد
- **السبب:** استخدام نموذج ثقيل، أو ضعف مواصفات الجهاز، أو طول النص المدخل.
- **الحل:** استخدم نموذج Gemma E2B، حاول اختصار الأسئلة، استخدم مستندات أصغر، وأغلق تطبيقات الخلفية.

### 3. تعذر تحليل ملف PDF
- **السبب:** الملف كبير جدًا، أو الصفحات غير واضحة (ممسوحة ضوئيًا كصور)، أو لا يحتوي على نص قابل للتحديد.
- **الحل:** جرّب ملف PDF أصغر، تأكد من وضوح الصفحات. ملاحظة: تحليل OCR قد يقتصر على الصفحات الأولى فقط.

### 4. ميزة "اسأل عن الصورة" لا تصف الصورة بصريًا
- **السبب:** بدون نموذج FastVLM، يعتمد نبض على استخراج النص (OCR) فقط.
- **الحل:** استورد نموذج الرؤية الاختياري FastVLM، أو استخدم صورًا تحتوي على نصوص واضحة ليتمكن نبض من قراءتها.

### 5. البحث الدلالي (Semantic Search) لا يعمل
- **السبب:** نموذج التضمين غير مستورد، أو المستند لم يتم فهرسته دلاليًا.
- **الحل:** استورد نموذج التضمين (Embedding Model)، أنشئ الفهرس الدلالي للمستند، واستخدم أدوات تشخيص RAG للتأكد من الحالة.

### 6. الخريطة لا تفتح
- **السبب:** لا يوجد تطبيق خرائط مثبت على الجهاز.
- **الحل:** تأكد من تثبيت تطبيق خرائط جوجل (Google Maps) أو أي تطبيق خرائط متوافق.

### 7. لماذا النماذج غير مدمجة في التطبيق؟
- **التوضيح:** لا يتم دمج النماذج داخل ملف APK بسبب حجمها الكبير (عدة جيجابايت) ولقيود التراخيص. يجب على المستخدم استيرادها يدويًا لمرة واحدة.

### 8. لماذا لا يتذكر نبض معلومة؟
- **السبب:** ذاكرة نبض معطلة، أو لم يتم طلب الحفظ بصيغة واضحة.
- **الحل:** إذا لم يتذكر نبض معلومة، تأكد أن ذاكرة نبض مفعلة وأنك طلبت الحفظ بصيغة واضحة مثل: تذكر أن اسمي عمار.
